## 轮臂增量遥操作说明

### 0. 轮臂遥操作特有参数

#### 0.1 参数说明

下表列出了增量遥操作相关参数及其默认值（前三项目前已由控制器服务自动配置，无需手动修改配置文件）：

---
| 参数 | 配置位置 | 默认值 | 遥操作必须配置值 | 配置方式 |
|---|---|---|---|---|
| `vrArmAccelTask.useVrArmAccelTask` | `task.info` | `false` | `true` | 自动（服务） |
| `armTrajInterpKinematicLimit.enable` | `task.info` | `false` | `true` | 自动（服务） |
| `use_vr_arm_kpkd` | 运行时 | `false` | `true` | 自动（服务） |
| `use_cpp_incremental_ik` | `launch_quest3_ik.launch` | `false` | `true` | launch 参数 |
| `wheel_ik` | `launch_quest3_ik.launch` | 按机器人版本自动 | `true` | launch 参数 |
---

#### 0.2 自动配置原理

启动 IK launch 时传入 `use_cpp_incremental_ik:=true wheel_ik:=true`，`wheel_ik_ros_uni_cpp_node` 在完成自身初始化后，会依次等待并调用以下三个控制器 ROS 服务完成参数配置：

| 服务名 | 作用 |
|---|---|
| `/enable_vr_arm_kpkd` | 切换手臂 PD 增益为 VR 专用值 |
| `/enable_vr_arm_accel_task` | 启用 vrArmAccelTask（VR 专用手臂加速度任务） |
| `/enable_arm_traj_interpolator` | 启用手臂轨迹插补增强路径 |

### 1. 启动注意事项

- 启动前请将 `ip_address` 替换为实际设备 IP。
- 控制器（终端 1）必须先启动并就绪，IK 节点（终端 2）会自动等待服务可用后再配置。

### 2. VR 软件端（Quest 3）

- 🚨 连接要求：Quest 3 与控制端必须使用**有线连接**。
- 稳定版本：`0.0.1-181-g2b295f2`（可在 VR 左手柄查看版本）。

### 3. 启动命令

#### 3.1 实物轮臂 + VR 增量遥操作（双终端）

1) 终端 1：

```bash
roslaunch humanoid_controllers load_kuavo_real_wheel.launch
```

2) 终端 2：

```bash
roslaunch noitom_hi5_hand_udp_python launch_quest3_ik.launch \
  ip_address:=xxx.xxx.xxx.xxx \
  use_cpp_incremental_ik:=true \
  use_incremental_hand_orientation:=false \
  wheel_ik:=true
```

终端 2 启动后，`ik_ros_uni_cpp_node` 初始化完成后会自动调用控制器服务完成配置，日志示例：

```
[wheel_ik] Configuring VR incremental mode via controller services...
[wheel_ik] /enable_vr_arm_kpkd set to true -> OK
[wheel_ik] /enable_vr_arm_accel_task set to true -> OK
[wheel_ik] /enable_arm_traj_interpolator set to true -> OK
[wheel_ik] VR incremental mode configuration done.
```

#### 3.2 仿真 + VR 增量遥操作（双终端）

1) 终端 1：

```bash
roslaunch humanoid_controllers load_kuavo_mujoco_sim_wheel.launch
```

2) 终端 2（同实物）：

```bash
roslaunch noitom_hi5_hand_udp_python launch_quest3_ik.launch \
  ip_address:=xxx.xxx.xxx.xxx \
  use_cpp_incremental_ik:=true \
  use_incremental_hand_orientation:=false \
  wheel_ik:=true
```

### 4. 参数说明

- `ip_address`：Quest3/遥操作链路使用的目标 IP 地址。
- `use_cpp_incremental_ik`：是否启用 C++ 增量 IK（建议 `true`）。
- `use_incremental_hand_orientation`：是否使用手部姿态增量控制。
- `wheel_ik`：是否启用轮臂 IK 相关控制链路。

#### 4.1 参数组合速查

| `use_cpp_incremental_ik` | `use_incremental_hand_orientation` | 位置模式 | 姿态模式 | 说明 |
|---|---|---|---|---|
| `true` | `false` | 增量 ✅ | 绝对 ⭕ | 位置增量 ✅ + 姿态绝对 ⭕ |
| `true` | `true` | 增量 ✅ | 增量 ✅ | 位置增量 ✅ + 姿态增量 ✅（全增量） |
| `false` | `false` | 绝对 ⭕ | 绝对 ⭕ | **无效组合**（当前轮臂不支持） |
| `false` | `true` | 绝对 ⭕ | 增量 ⭕ | **无效组合** |

#### 4.2 常用控制参数（建议直接改 launch 默认值）

如需长期使用固定灵敏度，建议直接修改 `src/manipulation_nodes/noitom_hi5_hand_udp_python/launch/launch_quest3_ik.launch` 中 `delta_scale_x/y/z` 的默认值：

```xml
<arg name="delta_scale_x" default="1.26" />
<arg name="delta_scale_y" default="1.26" />
<arg name="delta_scale_z" default="1.26" />
```

修改后，按常规启动命令即可生效，无需每次在命令行重复传参。

### 5. 手柄操作要点

| 操作项 | 触发方式 | 行为说明 |
|---|---|---|
| 激活遥操作 | `X + A` | 进入遥操作激活模式；建议等待约 5 秒后再开始手部控制。 |
| 复位与再次激活 | `X + A` | 执行 `X + A` 可复位；下一次使用仍通过 `X + A` 激活。 |
| 腰部总开关（X 组合） | 按下 `X` + 右摇杆上/下推并保持 | `X + ⬇ + 保持 = 🔒 关闭腰部位置+姿态跟随`；`X + ⬆ + 保持 = 🔓 开启腰部位置+姿态跟随`。 |
| 腰部位置跟随细分（非 X） | 未按 `X` 时，右摇杆上/下推并保持 | `⬇ + 保持 = 关闭腰部位置跟随（仍可转动腰部）`；`⬆ + 保持 = 恢复腰部位置跟随`。 |
| 激活左右手控制 | 左/右侧扳机 | 左扳机激活左手跟踪，右扳机激活右手跟踪；长按可持续控制对应手臂。 |
| 增量跟随行为 | 松开并再次按下侧扳机 | 松开后该侧机械臂保持当前位置；人手可自由调整，再次按下后从当前位置继续增量跟随（类似"推鼠标"）。 |
| 底盘控制 | 遥操作解锁 + 左/右摇杆 | 仅在遥操作已解锁时生效；左摇杆控制全向移动（前后、左右平移），右摇杆控制原地旋转。 |
| 腰部升降 | 侧扳机按下状态 + 升降操作 | 腰部升降为增量控制，需在侧扳机按下状态下进行。 |
| 上肢/腰部姿态跟随 | 人体动作实时输入 | 上肢俯仰与腰部转身为绝对姿态，实时跟随人体动作。 |
| 默认行为 | 轮臂激活后 | 无论是否按左右侧扳机，系统默认跟随人体腰部旋转。 |

> **生效前提**：仅当"腰部总开关（X 组合）"处于开启状态时，"腰部位置跟随细分（非 X）"逻辑才会生效。  
> **腰部功能速记**：`X + ⬇ = 关位置+姿态`，`X + ⬆ = 开位置+姿态`；未按 `X` 时，`⬇ = 仅关位置（可转腰）`，`⬆ = 恢复位置`。
