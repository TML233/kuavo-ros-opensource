# 轮臂手臂期望力控制说明

本文档介绍 Kuavo 轮臂机器人手臂末端期望力控制的使用方法，包括期望力设置、仿真外力施加、挥空检测、力估计查看以及参数动态调节等功能。

## 目录

### 相关文件目录

- `src/humanoid-wheel-control/humanoid_wheel_interface/config/kuavo_s61/task.info` — 接触力相关配置参数
- `scripts/ee_force_control_cli.py` — 终端交互式期望力控制脚本
- `scripts/ee_force_control_gui.py` — GUI 期望力控制脚本

### 演示脚本启动方式

**终端交互式控制（CLI）：**
```bash
python scripts/ee_force_control_cli.py
```
启动后进入终端菜单，支持以下操作：输入期望力（选择左/右/双手，分别输入 x、y、z 方向力值，单位 kg，自动转换为 N）、期望力归零、快速启动（双手 z 方向 -3kg）。

**GUI 滑条控制：**
```bash
python scripts/ee_force_control_gui.py
```
启动后弹出 PyQt5 窗口，通过滑条分别调节左右手 x、y、z 三方向的期望力（单位 kg），并包含挥空检测开关按钮。需要图形界面环境支持。

**施加期望力脚本1**
```bash
python3 src/demo/test_kuavo_wheel_real/armContactForce/cmd_arm_force_test.py
```
该脚本适合直接修改源码中的固定参数后运行，不通过命令行传参。主要配置项如下：

- `HAND`：选择施力对象，可选 `left`、`right`、`both`
- `LEFT_FORCE`：左手期望力，格式为 `(x, y, z)`，单位 N
- `RIGHT_FORCE`：右手期望力，格式为 `(x, y, z)`，单位 N
- `ENABLE_EXTERNAL_FORCE`：是否同步发送与期望力同等大小的 `/external_wrench/*`
- `DISABLE_FORCE_EMPTY_DETACT`：施力前是否先关闭挥空检测，结束后自动恢复

运行后会先发送设定好的期望力；若启用了 `ENABLE_EXTERNAL_FORCE`，还会同步发送同等大小的仿真额外外力。终端提示：

`已施加末端力，按回车撤销...`

按回车后会自动清零期望力；若此前开启了额外外力或关闭了挥空检测，也会一并恢复。

### 施加期望力脚本2

新增脚本：`src/kuavo_humanoid_sdk/kuavo_humanoid_sdk/kuavo_strategy_pytree/pick_place_box/case_wheel_arm_force.py`

#### 关键可配置参数

- `BOX_WEIGHT_KG`：期望力对应重量（kg），默认 `6.0`
- `INTERPOLATION_SPEED`：期望力插值速度（N/s），默认 `2000.0`
- `ARM_MOVE_TIME`：手臂移动到目标位姿的时间（s），默认 `2.0`
- `FORCE_EMPTY_DETECT_ENABLE`：执行前是否保持挥空检测，默认 `False`（即先关闭）
- `ARM_EE_POSE_DEG`：左右臂目标位姿，格式为 `[x, y, z, yaw, pitch, roll]`（角度制）

#### 运行方式

在工作空间根目录执行：

```bash
python src/kuavo_humanoid_sdk/kuavo_humanoid_sdk/kuavo_strategy_pytree/pick_place_box/case_wheel_arm_force.py
```

运行日志会输出当前配置的期望力重量、插值速度和手臂运动时间。施加期望力后，终端提示：

`期望力已施加，按回车撤销...`

按回车后会自动撤销期望力并恢复挥空检测。

#### 注意事项

- 请先确认机器人处于安全状态，目标点位无碰撞风险
- 调整 `BOX_WEIGHT_KG` 时建议从小到大逐步验证
- 若需要更平滑的力变化，可适当降低 `INTERPOLATION_SPEED`
- `ARM_EE_POSE_DEG` 的姿态输入为角度，脚本内部会自动转换为弧度

## 配置参数（`contact_force_params`）

配置文件路径：`src/humanoid-wheel-control/humanoid_wheel_interface/config/kuavo_s61/task.info`

```
contact_force_params
{
  contactForceErrorThreshold  0.15    ; 误差阈值（rad），超过此值禁用接触力任务
  weightArmAccelReduced    30.0    ; 施加期望力时的手臂任务权重
  transitionTime   0.3    ; 权重平滑过渡时间常数（秒）
  forceThreshold           5.0    ; 期望力阈值（N），超过此值认为施加了期望力
  interpolationSpeed       300.0   ; 期望力插值速度（N/s）
}
```

**参数说明：**

| 参数名 | 默认值 | 单位 | 说明 |
|--------|--------|------|------|
| `contactForceErrorThreshold` | 0.15 | rad | 关节误差阈值。开启挥空检测后，若施加期望力时手臂关节最大误差超过此值，则认为手臂挥空，自动撤销期望力 |
| `weightArmAccelReduced` | 30.0 | — | 施加期望力时 WBC 中手臂加速度任务的权重值 |
| `transitionTime` | 0.3 | s | 施加/撤销期望力时，手臂任务权重的平滑过渡时间常数 |
| `forceThreshold` | 5.0 | N | 期望力生效阈值。期望力大于此值时才认为正在施加期望力，触发权重切换等逻辑 |
| `interpolationSpeed` | 300.0 | N/s | 期望力插值速度。控制器内部对期望力进行线性插值平滑，此值决定力从当前值到目标值的变化速率 |

## ROS 话题接口

### 发布话题 (Published Topics)

#### 1. `/desired_ee_force/left` 和 `/desired_ee_force/right` - 设置手臂末端期望力

**消息类型：** `geometry_msgs/WrenchStamped`

**功能描述：** 分别设置左手和右手末端的期望接触力。控制器接收后，通过 WBC 在手臂末端施加对应的力。

**注意事项：**

1. 力的方向以机器人本体坐标系（`base_link`）为参考；
2. 力的单位为牛顿（N），最大限制为 60N；
3. `wrench.torque` 通常置零，仅使用 `wrench.force` 部分；
4. 发送全零力即撤销期望力；

**字段说明：**
```yaml
std_msgs/Header header:
  uint32 seq              
  time stamp              
  string frame_id         

geometry_msgs/Wrench wrench:
  geometry_msgs/Vector3 force:
    float64 x             # X方向期望力（N），正值向前
    float64 y             # Y方向期望力（N），正值向左
    float64 z             # Z方向期望力（N），正值向上
  geometry_msgs/Vector3 torque:
    float64 x             # 通常为 0.0
    float64 y             # 通常为 0.0
    float64 z             # 通常为 0.0
```

**快速测试命令：**

左手施加 -z 方向 29.4N（约 3kg）的期望力：
```bash
rostopic pub /desired_ee_force/left geometry_msgs/WrenchStamped "header:
  seq: 0
  stamp:
    secs: 0
    nsecs: 0
  frame_id: ''
wrench:
  force:
    x: 0.0
    y: 0.0
    z: -29.4
  torque:
    x: 0.0
    y: 0.0
    z: 0.0"
```

**相关演示脚本：**
- `scripts/ee_force_control_cli.py` — 终端交互式控制，支持选择左/右/双手、输入 xyz 三方向力值（以 kg 输入，自动转换为 N）
- `scripts/ee_force_control_gui.py` — GUI 滑条控制

#### 2. `/external_wrench/left_hand` 和 `/external_wrench/right_hand` - 仿真施加外力

**消息类型：** `geometry_msgs/Wrench`

**功能描述：** 在仿真环境中（MuJoCo / Gazebo）给机器人手臂末端施加额外外力，用于模拟外部接触力或负载。

**注意事项：**

1. 仅在仿真环境下有效，实物环境无需使用；
2. 消息类型为 `Wrench`（无 Header），与期望力话题的 `WrenchStamped` 不同；

**字段说明：**
```yaml
geometry_msgs/Vector3 force:
  float64 x               # X方向外力（N）
  float64 y               # Y方向外力（N）
  float64 z               # Z方向外力（N）

geometry_msgs/Vector3 torque:
  float64 x               # X轴外力矩（Nm）
  float64 y               # Y轴外力矩（Nm）
  float64 z               # Z轴外力矩（Nm）
```

**快速测试命令：**

给仿真中左手末端施加 -z 方向 30N 的外力：
```bash
rostopic pub /external_wrench/left_hand geometry_msgs/Wrench "force:
  x: 0.0
  y: 0.0
  z: -30
torque:
  x: 0.0
  y: 0.0
  z: 0.0"
```

#### 3. `/enable_force_empty_detact` - 挥空检测开关

**消息类型：** `std_msgs/Bool`

**功能描述：** 开启或关闭手臂挥空检测功能。开启后，若施加期望力时检测到手臂关节最大误差超过 `contactForceErrorThreshold`（默认 0.15 rad），则认为手臂挥空（未接触到物体），自动撤销期望力以保护机器人。

**注意事项：**

1. `true` 为启用挥空检测，`false` 为关闭；
2. 挥空判定条件：施加期望力 + 关节误差超过阈值；
3. 触发挥空后会自动清除所有期望力；

**字段说明：**
```yaml
bool data    # true: 启用挥空检测；false: 关闭挥空检测
```

**快速测试命令：**

启用挥空检测：
```bash
rostopic pub /enable_force_empty_detact std_msgs/Bool "data: true"
```

关闭挥空检测：
```bash
rostopic pub /enable_force_empty_detact std_msgs/Bool "data: false"
```

**相关演示脚本：** `scripts/ee_force_control_gui.py`（包含挥空检测开关按钮）

#### 4. `arm_contact_force_debug/left_arm_force` 和 `right_arm_force` - 手臂力估计数据

**完整话题名：**
- `/humanoid_wheel/arm_contact_force_debug/left_arm_force`
- `/humanoid_wheel/arm_contact_force_debug/right_arm_force`

**消息类型：** `std_msgs/Float64MultiArray`

**功能描述：** 发布手臂末端的接触力估计数据，用于调试和分析手臂与环境的交互力。

**字段说明：**
```yaml
Float64MultiArray data    # 6维向量：[fx, fy, fz, tx, ty, tz]
                          # fx, fy, fz — 末端接触力估计（N）
                          # tx, ty, tz — 末端接触力矩估计（Nm）
```

**快速查看命令：**

查看左臂力估计：
```bash
rostopic echo /humanoid_wheel/arm_contact_force_debug/left_arm_force
```

查看右臂力估计：
```bash
rostopic echo /humanoid_wheel/arm_contact_force_debug/right_arm_force
```

## ROS 服务接口

### 调用服务 (Called Services)

#### 5. `/set_contact_force_params` - 动态修改接触力参数

**服务类型：** `kuavo_msgs/setContactForceInterpParams`

**功能描述：** 在运行时动态修改接触力控制的两个关键参数：权重过渡时间（`transitionTime`）和期望力插值速度（`interpolationSpeed`），无需重启控制器。

**请求字段：**
```yaml
float64 transition_time       # 权重平滑过渡时间常数（秒），必须为正数
float64 interpolation_speed   # 期望力插值速度（N/s），必须为正数
```

**响应字段：**
```yaml
bool success      # 操作是否成功
string message    # 状态消息
```

**快速测试命令：**

将过渡时间设为 0.5s，插值速度设为 200 N/s：
```bash
rosservice call /set_contact_force_params "transition_time: 0.5
interpolation_speed: 5.0"
```
